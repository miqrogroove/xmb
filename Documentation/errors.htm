We hope that you never have any problems with this software, but we realize that occasionally you will encounter a few glitches or normal error messages. We have compiled a small list of common problems you may encounter and some suggested solutions for these problems.

If your particular issue is not addressed below please don't hesitate to post your questions to our Support Team at http://forums.xmbforum.com/

1) "Help! I can't log into my new boards!"


This is a common issue with new installs and is usually due to the $full_url not being filled our correctly in your config.php file.

Open your config.php file in a text editor and find the *// Path-settings* section. Within that section you will see

$full_url = 'FULLURL';

You need to make sure to replace the "FULLURL" with the url to your forums - and don't forget the trailing forward slash. For example,

$full_url = 'http://forums.xmbforum.com/';

Save the config.php file and upload to your server. 99% of the time this will clear up any "new install" login issues.

"I get a Column count not found at row 1 error!"


This is due to hacks. XMB 1.9.5 will NEVER display this message as all SQL queries it issues are fully specified. If you have hacks installed, you will need to ask for support from the person you obtained the hack from.

However, the best bet to resolve this issue is to return XMB 1.9.5 to pristine condition by performing an upgrade. Just follow the steps described in the upgrade documentation. The upgrade basically restores your board to correct working condition.

"A few of my new members are getting an error message and they are not being allowed to register! The error message is 'Access to this website is currently not possible as your hostname/IP appears suspicious' - what is that all about?!


The error message "Access to this website is currently not possible as your hostname/IP appears suspicious" is due to a "check" installed by default to prevent any suspicious activity that could harm your forums. Sometimes it can get in the way of valid registrations. This is particularly true in countries like Australia where all the major ISPs use transparent proxies. If many users get this message, just turn the check off.

You can turn off this check by opening your config.php file in a text editor. Find the "// Registration settings" section and change $ipcheck to "off" instead of the default "on". Save the config.php file and upload to your server.

3) "I tried to register a new account for my forum but I get this error: 'Error: You have already registered today, you can re-register again in 24 hours' - why is this happening?!


This is due to a check in the config.php file. You can turn of the check by opening your config.php file in a text editor and looking in the "// Registration settings" section and changing $ipreg to "off" instead of the default "on". Save the config.php file and upload to your server.
