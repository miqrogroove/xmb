Installation

Suitable for most operating systems including: Windows, MacOS X, FreeBSD, NetBSD, OpenBSD and Linux

Please review the Prerequisites before you begin.

Most web hosts run a version of Unix, such as Solaris or AIX, or a freely available versions such as FreeBSD (such as Yahoo and Pair Networks) or Linux (such as Aventure Media or EV1). As long as your hoster runs the required versions of MySQL and PHP, there’s no minimum or recommended operating system version.

The basic installation process is as follows:

  • Create a database for your forum
  • Upload all files
  • Do one of:
    • Locally edit config.php and upload to your host
    • Use the installation script to configure your config.php (may not work due to hoster's permissions) or
    • Use the installation script to generate a config.php and upload to your host
  • Run the installation script and follow the instructions
  • Logon and customize your forum

Let’s go through these steps in detail.

Create the database

Make sure that you have enough web space for your forum, and that your hosting account includes both PHP and MySQL support.

Create a database for your forums, using the MySQL Management Tool from your site's Control Panel or by using MySQL's administration tool directly. When you are creating the database, note the database name ($dbname), username ($dbuser) and password ($dbpw).

In the following example, we will use the mysql command line tool to create the database. You should create a name and use usernames and passwords which are different, or else you will have a security issue.

mysql> CREATE DATABASE xmb19;

Query OK, 1 rows affected (0.03 sec)

mysql> GRANT ALL PRIVILEGES ON xmb19.* TO 'xmbuser'@'localhost' IDENTIFIED BY 'foo123';

Query OK, 0 rows affected (0.03 sec)

In the above example case, $dbname will be xmb19, $dbuser is xmbuser, and $dbpw is foo123. In addition, we have restricted database access to the localhost, which is fine if the database server is on the same host as the web server. If it is not the same, you should put in the fully qualified domain name of your web server in place of localhost.

Upload

Create a folder for your forum in your web space. How you do this is dependant upon your web host, but we will demonstrate here using cPanel, as it is very common:

Open the cPanel File Manager, navigate inside your public_html folder by clicking on the little folder icon on the left of “public_html”, and then click “Create New Folder”. You can use any name, but most people use “forum”.

Whatever folder name you use is part of the $full_url variable, so if your site is located at www.example.com, and the name for the folder you created is “forum”, the value of $full_url will be http://www.example.com/forum/

If you want, you can simply use the top level of your account, but this means your account will display the forum by default, which may not be what you had in mind.

Alternatives to cPanel include using Frontpage (WebDAV), ftp’s “mkdir” command or logging on directly to your host using telnet or ssh and using Unix’s “mkdir” command.

Upload all the files from the distribution to your site. Everything in the "Files" folder requires uploading, including all sub-folders and files.

Figure 1: Files to be uploaded

You should use “binary” to ensure that the XMB is correctly transferred to your host. Most FTP clients do this automatically for you, but if you are using a command line ftp client, use the “bin” command prior to transferring.

You must create each of the folders and upload each of the files within the “db”, “images”, “include”, “install” and “lang” folders or else XMB will not function.

Configure config.php

There are two methods of installing XMB: configuring config.php manually or automatically. There are pros and cons for each. Experienced users will probably prefer to use the manual method and first time administrators will probably prefer to use the automatic method.

Manual configuration

Edit config.php in your favorite text editor. The only fields which MUST be changed are:

$dbname MySQL database name
$dbuser MySQL database user
$dbpasswd MySQL database password
$full_url The full URL to your forum, including a trailing slash
$tablepre The table prefixes. Normally xmb_

 M You MUST make sure that the last line and characters in config.php are ?>

Do not add any lines or convert this file to DOS format or XMB will not work.

Once you have edited config.php to your satisfaction, upload it (in binary mode if using ftp), over-writing the previous copy of config.php.

Skip ahead to the installation section.

Automatic configuration

Automatic configuration requires a temporary weakening of file permissions (security) during installation on most systems.

There are many ways to achieve this. In many graphical FTP clients, you right click on a file and choose chmod or Change Attributes or something similar. Then check all check marks in the permissions area pertaining to write or read, or enter 666 where it shows a textbox with the numbers: 644. This represents:

user: read/write

group: read/write

other: read/write

You may need to do this in your host's CPanel or via shell access. If you can't work it out, ask your host for help.

N With file permissions set to world-writable (mode 666), anyone on the Internet could take over your forum, so make sure you check or change the permissions back to 644 after you have installed XMB.

XMB’s installer tries to automatically change the file permissions of config.php after installation has finished, but some security conscious hosts do not allow this functionality, so it does not always work. All XMB files should be mode 644 once installation is complete.

Installation

G Please note that the XMB 1.9.2 (Nexus) installation method is different from older versions, including beta versions of 1.9.

Start your web browser

Input the URL where your forum will be located and then add /install/index.php to the end. Example:

http://www.example.com/forums/install/index.php

If you have typed the installation URL correctly, you will see the following image:

Click on it to begin installation.

The second page is the XMB Version Check. This is used to show you the most recent build of XMB Forum. If your version is earlier, go get a later version as it'll be more secure and contain fewer bugs.

However, if you have the latest version, click on the 'Install XMB 1.9.2' button to proceed.

Read and understand the XMB 1.9.2 Nexus License Agreement, which is the GNU GPL version 2. If you're prepared to accept it, 'I agree to terms' button to proceed.

Next up is the Configuration File Setup Page. This is the most important step. Input the required database name, username, password from above where you configured the database.

The only other mandatory field is "Full URL". The Full URL helps the forum base itself relative to wherever you need it, and allows you to move XMB easily between hosters if you need it.

If your forum is going to be at

http://www.example.com/forum/

then, that's the full URL. Simple! Remember to put a slash at the end.

The other settings are optional, but you may activate them if you wish. The most important to note is "ipreg". This is not a useful setting in many countries where ISPs use transparent proxies. Leaving this in the "on" state stops more than a couple of registrations per day.

Once finished with the configuration choices, click on the 'Configure' button to proceed. If the file permissions on config.php are correct, XMB will have written out a correct config.php.

You need to set an Super Administrator password. This is a new feature since earlier versions of XMB.

Do not choose a name like "admin" or "Administrator" or "root". Be creative. The password be complex - more than 8 characters long, and contain numbers and punctuation marks.

The installer automatically installs XMB 1.9.2 Nexus on to your database.

Scroll down to the end of the page, and click on 'here' to redirect to your forums.

M If you have WARNINGS from the Installer, it will tell you why it has stopped the installation process. Contact your host if it has issues related to PHP or mySQL versions.

Post-installation Checks

  • Make sure the file permissions of all files, including config.php, are mode 644 (or user: read/write, group and other: read only).
  • Directories should be mode 755 (or user: read/write/execute, group and other: read and execute), but can also be 711 (or user: read/write/execute, group and other: execute only)
  • If it still exists, delete the /install/ folder using FTP, shell or your host’s control panel.

Customizing Your Forum

Now that you have successfully installed XMB, you can make it your own.

You can modify or create a new theme, or add what are termed “hacks” to your forum to provide additional functionality.
